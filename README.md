# タスク管理アプリ

考えずに行動に移すための、シンプルなタスク管理アプリです。
タスクを「思いつき / 今日やる / 明日やる」に分けることで、
**やることを溜めすぎず、今やるべきことに集中できる設計**にしています。

※ 初回起動時は、動作確認用のサンプルタスクが表示されます。

---

## デモ

[https://nene25dev-todo-app.netlify.app/](https://nene25dev-todo-app.netlify.app/)

---

## 主な機能

* タスクの追加 / 編集 / 削除 / 復元
* タスクの分類（思いつき / 今日やる / 明日やる）
* ドラッグ＆ドロップによるタスク移動
* 完了したタスク・ごみ箱の管理
* フィルター切り替え（全て / 完了 / 削除）
* データベースへの永続化（Prisma + SQLite）

---

## 制作意図・設計方針

このアプリは、
「やることを考えること自体が負担になる」状態を減らすことを目的に設計しました。

* **今日やるタスクは最大3件**
  → 優先順位を強制的に意識できる
* **1タスクあたり最大30分**
  → タスクを細分化し、着手しやすくする
* **ドラッグ操作中心のUI**
  → 考えずに直感的に整理できる

タスク管理を「管理」ではなく
**行動を後押しする仕組み**として捉えています。

## 実装上の工夫

* 同じ構造のセクション表示が繰り返されていたため、セクション情報を配列で管理し、map とコンポーネント化によって共通化しました。
これにより、表示ロジックを整理し、レイアウト変更や仕様追加時の修正コストを下げています。
* バリデーション処理を関数としてまとめ、入力項目が増えた場合でも再利用しやすい構造に整理しました。
* タスク移動処理は共通ロジックに集約し、並び替えと分類変更を同じ流れで扱うことで、実装の重複を減らしました。
* フロントエンドとバックエンドで型を共有するため shared/types を作成し、TypeScript による型安全なデータ連携を実現しました。
* UI設計の検討において AI ツールを補助的に使用していますが、構造設計・ロジック実装は自身で行っています。
