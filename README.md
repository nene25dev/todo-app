# タスク管理アプリ

考えずに行動に移すための、シンプルなタスク管理アプリです。
タスクを「思いつき / 今日やる / 明日やる」に分けることで、
**やることを溜めすぎず、今やるべきことに集中できる設計**にしています。

※ 初回起動時は、動作確認用のサンプルタスクが表示されます。

---

## 主な機能

* タスクの追加 / 編集 / 削除 / 復元
* タスクの分類（思いつき / 今日やる / 明日やる）
* ドラッグ＆ドロップによるタスク移動
* 完了したタスク・ごみ箱の管理
* フィルター切り替え（全て / 完了 / 削除）
* データベースへの永続化（Prisma + SQLite）

---

## 制作意図・設計方針

このアプリは、
「やることを考えること自体が負担になる」状態を減らすことを目的に設計しました。

* **今日やるタスクは最大3件**
  → 優先順位を強制的に意識できる
* **1タスクあたり最大30分**
  → タスクを細分化し、着手しやすくする
* **ドラッグ操作中心のUI**
  → 考えずに直感的に整理できる

タスク管理を「管理」ではなく
**行動を後押しする仕組み**として捉えています。

## 実装上の工夫

- **フロントエンドとバックエンドの分離**  
  フロントエンド（React）とバックエンド（Node.js + Express）を分けて構成し、REST APIを通してデータをやり取りしています。  
  UIの処理とサーバー側のデータ処理を分離することで、役割を明確にし、保守しやすい構成にしています。

- **データの永続化**  
  PostgreSQL と Prisma ORM を使用してタスクデータをデータベースに保存しています。
  これにより、ページを再読み込みしてもタスクの状態が保持されます。

- **ドラッグ＆ドロップとデータ更新の連携**  
  タスクの並び替えや分類変更をドラッグ＆ドロップで操作できるようにしました。  
  操作後の状態はAPIを通してデータベースに保存し、UIとデータの状態が一致するようにしています。

- **型共有による安全なデータ通信**  
  `shared/types` ディレクトリを作成し、フロントエンドとバックエンドで型を共有しています。  
  TypeScriptを利用することで、API通信時のデータの不整合を防いでいます。
